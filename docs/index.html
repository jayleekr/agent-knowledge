What Happens When an Agent Learns Something?

Every mistake or discovery becomes a lesson for the whole team.

1. Something goes wrong

Herald gives a creator incorrect formatting advice.

2. The agent fixes it

Herald corrects the advice and notes what happened.

3. Writes it down

The lesson is recorded in the agent's personal lesson file.

4. Shared with the team

Pushed to GitHub so all agents can see it.

5. Kaizen spots a pattern

"Three agents made similar mistakes this week ‚Äî let's fix the root cause."

üí° Real Example

Herald told a creator to use author: "Name" in their article, but the correct field is creator: "Name". Herald recorded this mistake. Kaizen later noticed the guide template itself was wrong, and proposed updating it ‚Äî preventing the same mistake for all future creators.

Self-Evolving Quality Standards

Kaizen doesn't just follow rules ‚Äî it improves the rules themselves.

Check each quality metric

Is "Creator Satisfaction" still meaningful? Is the threshold right?

Remove what doesn't work

If a metric never produces useful insights, disable it.

Add what's missing

Noticed a new problem pattern? Create a new metric to track it.

Adjust thresholds

As quality improves, raise the bar. If it's too strict, lower it.

üí° Example

Kaizen started with 10 quality criteria. After a week, it noticed "Pipeline Throughput" was always at zero (no submissions yet), so it temporarily disabled it. Meanwhile, it added a new criterion: "Onboarding Completion Rate" because three new creators dropped off during signup.
